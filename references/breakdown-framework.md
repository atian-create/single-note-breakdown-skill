# Breakdown Framework

## 1. Post Type

Name the visible format:

- image note
- long graphic note
- talking-head video
- tutorial video
- product note
- emotional story
- comparison
- checklist
- interaction post

The format changes the mechanism. A tutorial earns saves differently from an
emotional story. A product note needs proof. An interaction post needs a low-bar
comment trigger.

## 2. Click Reason

Look for:

- title keyword
- cover promise
- visible proof
- emotional tension
- identity fit
- curiosity gap
- result shown before explanation

Useful sentence:

> Users click because the post makes [specific audience] believe they will get
> [specific result / answer / recognition] quickly.

## 3. Read-Through Reason

Look for:

- opening sentence
- story sequence
- step-by-step structure
- suspense
- concrete scene
- repeated "this is about me" feeling
- useful details appearing early enough

## 4. Save / Share Reason

Look for:

- checklist
- template
- step
- comparison table
- repeatable method
- place / product / tool list
- sentence users want to remember

## 5. Comment Demand

Read comments when available. Classify demand:

- asking for details
- asking for links or tools
- sharing similar experience
- disagreement
- asking if it applies to them
- requesting a template
- asking for the next part

## 6. Title And Cover Mechanism

Separate title and cover:

- Title: what question or promise does it create?
- Cover: what visual proof or keyword does it show in one second?
- Together: do they make the same promise or two conflicting promises?

## 7. Structure Map

Map the post into 4-7 beats:

1. hook
2. problem / scene
3. evidence / story
4. method / list
5. proof / example
6. conclusion
7. comment or next action

## 8. Learnable Parts

Pick 1-3 parts ordinary creators can actually use:

- opening move
- title angle
- cover promise
- section order
- proof type
- comment trigger
- saveable template

## 9. Non-Copyable Parts

Flag what should not be copied:

- creator identity
- private experience
- rare resource
- original data
- personal image
- brand trust
- expensive production
- exact wording or story

## 10. Next-Post Adaptation

Do not give a huge idea list. Give one practical next-post direction:

- target audience
- angle
- title draft
- cover text
- 4-part outline
- what evidence the user needs
